# Contributing

Thanks for your interest in this project.

## Scope

This repository is intended to become a public educational project about the evolution of AI applications from Function Calling to Agent OS.

Public contributions should focus on:
- chapter content
- timeline quality
- concrete examples and interactions
- repository structure and reader experience

Internal writing workflows, private drafting notes, and personal publishing tooling are intentionally kept out of the public repo surface.

## Content Principles

All contributions should follow these rules:

1. Problem-driven first  
   Every concept should be introduced as a response to a concrete failure or limitation in the previous stage.

2. History matters  
   Concepts should be anchored to real events, papers, products, or protocol milestones whenever possible.

3. Product and systems view  
   Prefer explanations that help readers think like engineers building real systems, not just running demos.

4. Concrete over vague  
   Prefer examples, interactions, and data shapes that help readers build a real system intuition.

5. Clarity over coverage  
   The goal is not to collect every concept, but to preserve the causal chain.

## Repository Layout

- `README.md`: repository homepage and project positioning
- `TIMELINE.md`: key event timeline
- `ch00-llm-limitations/` ... `ch10-production/`: chapter directories

## Before Opening a PR

Please check:
- the change strengthens the causal narrative
- headings and structure are easy to scan
- examples are concrete, accurate, and easy to follow
- claims are either sourced, widely accepted, or clearly marked as opinion

## What Not to Add

Avoid:
- framework-specific tutorials with no narrative connection
- generic AI news dumps
- purely motivational prose
- private notes, vault-specific metadata, or personal workflow files
